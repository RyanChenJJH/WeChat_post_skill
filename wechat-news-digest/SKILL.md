# 微信公众号资讯速递 Skill

## 这个 skill 做什么
把公开资讯加工成**轻度资讯整理**型的公众号内容：事实摘要 + 轻度解读 + 风险提示，帮读者省时间。
默认产出**少而精深写推文**（精选 2-3 条，每条 600–900 字讲透，不灌水不凑数），
也支持多条大合集、单条轻解读和工具/公司动态简报。输入可带原文（无需联网）。

## 这个 skill 不做什么（定位红线，务必内化）
不写专家深度研报、不给投资建议、不做独家爆料、不做权威判断、不做夸张预测。

作者（用户）不具备深度行业咨询能力——所以**不靠 AI 编造"行业洞察"撑场面**。那种"必将颠覆""资本疯抢"
"这意味着行业格局彻底改变"的判断，既无人能背书、又最容易侵权和出事。本 skill 的价值来自
"把公开事实整理清楚 + 注明来源 + 标好风险"，不是来自观点的锋利。当你想加一句很"有洞察"的判断时，
先问：这是事实吗？有来源吗？错了谁负责？答不上来就删掉或降级成"值得继续观察"。

## 默认配置（用户没特别说明时直接采用，并在开头一句话注明假设）
- 账号方向：AI / 科技 / 商业资讯整理
- 读者：普通科技关注者、AI 工具用户、创业者、内容从业者
- 风格：通俗但克制、清晰、信息密度高；把专业翻成大白话、可打比方拉近距离，但不标题党、不绝对化、不浮夸
- 默认形态：少而精，精选 2-3 条深写（每条 600–900 字，中间讲透；导语点主线、不用速览）；要多条大合集或单条轻解读时另按对应模板
- 结论方式：谨慎表达，不下绝对判断
- 完整账号画像（定位/读者/栏目名/语气/排版主色/互动句式/AI 声明文案）见
  `references/account-profile.md`，用户可编辑；未编辑时用其默认值并注明"采用默认画像"

## 三条最高优先级铁律

### 铁律一 · 版权（最重要）
**读原文是为了提取事实，不是对着原文改写。**
流程必须是：打开链接 → 抠出硬事实（主体 / 动作 / 数字 / 时间 / 出处）记成要点 → **关掉原文** →
只对着自己的要点笔记成稿。绝不允许把原文段落和草稿放在一起逐句"翻译/替换近义词"——那就是洗稿，
是微信投诉和处罚最狠的一类。细则见 `references/copyright-guardrails.md`，写每一条之前都要遵守。
**（v3.0 补充）** 正文要写得通俗、能涨知识，但"丰富"必须来自**你自己的再解释**（用大白话重讲术语、
补背景、打比方、多源综合），**不是把原文那篇展开复述**。叙事越丰富，越要逐句跑量化回查（13 字重合）。
**（v3.1 补充·重要）** 当输入**自带原文**（`> [!quote]- 原文` 块）时，原文就在眼前，洗稿更"顺手"、风险更高：
深写每条 600–900 字都必须**逐句**回查（与原文无连续 ≥13 汉字重合）；英文原文**不许逐句直译当正文**，
同样从事实重写、用自己的话讲。版权是不可逾越的前提。

### 铁律二 · 事实
区分事实、推测、观点：有明确来源、时间、主体的才当事实；其余一律用"可能 / 或将 / 有望 /
这释放出一个信号"等不确定措辞。绝不编造数据、来源、人物评价、排名、案例。
抓取卡片里的"凸显了 / 标志着 / 意味着 / 有望成为"等是**待核实的解读线索**，不是可直接采用的事实。
细则见 `references/fact-risk-control.md`。

### 铁律三 · 标识与诚实
AI 生成内容**必须主动标识**：正文文末固定一句 AI 声明 + 发布时勾选公众平台「AI 内容声明」功能；
整理/编译类内容**不打"原创"标**，不冒充人工独立采写。依据《人工智能生成合成内容标识办法》
（2025-09-01 施行）及微信平台配套规则。缺声明在发布前检查中是 ❌ 硬项。细则见 `references/copyright-guardrails.md` 9.5。

## 标准工作流（详见 references/workflow.md）
1. **接收并解析输入**：原始资讯文本 / 资讯链接 / 抓取的 md 卡片 / **带原文的资讯 md**（每条含 `> [!quote]- 原文` 块——直接用原文，无需联网）
2. **选题评分 + 组稿（少而精）**：双门槛筛 + 三维评分，**精选 2-3 条深写**（输入已是预选 2-3 条则全做、不再筛）；
   定封面条目/主线；**不沿用上游 ⭐️**。见 `references/topic-selection.md`
3. **逐条事实提取**：**优先读输入里的原文**（`[!quote]` 块）；无原文且需核实时才联网。读完 → 关原文 → 凭笔记写。来源类型按 S/A/B/C 分级
4. **版权 + 事实风险判断**：套用三条铁律的细则文件，给每条打版权风险分和可信度分
5. **成稿 + 排版**：按**少而精模板**写 2-3 条深写正文(L1，每条 600–900 字，**导语点主线、无速览**)，再转微信排版稿(L2)；
   **深写每段逐句跑 Q1**（与原文无连续 ≥13 字重合）；不达标条目宁可不选，不强行写
6. **发布前检查 + 变现建议**：跑 `prepublish-checklist.md`（含量化回查、AI 声明硬项），再给 `monetization.md` 的轻量变现建议

## 输出形态（详见 references/digest-templates.md）
- **少而精深写（默认）**：封面总标题(主推) + 1–2 句导语点主线 + **2-3 条深写**(每条 600–900 字) + 结尾区(互动 + AI 声明 + 参考来源)。
  **不用本期速览、不用简讯区**。条目 = 轻标记(发生了什么/为什么值得关注/仍需注意/来源) + 连贯叙事主体(中间把机制/背景/影响/名词讲透、就地通俗解释)；"仍需注意"为条件字段
- **封面总标题**：默认用**主推** 1 个（可附 2-3 备选），区别于条目小标题（见 `references/titles-and-style.md`）
- **多条大合集（备选）**：用户明确要时才走——钩子式本期速览 + 头条/标准/简讯分层节奏
- **单条资讯轻解读**：挑 1 条重要的，800–1500 字
- **AI 工具 / 公司动态简报**：把同类资讯聚成一个专题
- **微信排版稿（L2）**：成稿后另产一份可直接粘贴进公众号编辑器的内联样式稿，
  由确定性转换生成（模型只产 markdown，不手写 HTML）。见 `references/wechat-formatting.md`

## 配图（v1）
**绝不使用原文的图片、图表、截图**（除非用户明确说明已获授权）。v1 只输出**文字版配图建议**
（封面图 / 正文图的描述），由用户自行用原创方式生成或选用可商用素材。

## 拒绝边界
如果用户要求洗稿、伪原创、去掉来源、规避查重/原创检测、冒充原创、盗用他人图片——**直接拒绝**，
并改为提供合规写法：从事实重写 + 注明来源 + 原创配图。这条不可协商。
