# 工作流细则

按以下 6 步执行。任何一步信息不足，最多向用户追问 3 个问题；用户说"直接写"则用默认配置并标注假设。

## Step 1 · 接收并解析输入

可能的四种输入：

1. **原始资讯文本**：用户直接粘贴的新闻/公告/纪要。
2. **资讯链接**：一条或多条 URL。联网可用时打开读取（遵守版权铁律：读完提取事实即关闭）。
3. **抓取的资讯卡片 md（仅摘要+链接）**：如 `20260609_AI人工智能.md`——已由 AI 抓取、打分、附背景和参考链接。
4. **带原文的资讯 md（新主力）**：如 `20260621_AI人工智能 (2).md`——每条在卡片下还带一个 `> [!quote]- 原文` 块（常为一手原文/英文原文）。
   **直接用原文块提取事实，无需联网。** 输入也可能**只含用户预选的 2-3 条**。

> ⚠️ 卡片里的摘要句**本身是 AI 写的解读**（"凸显了""标志着""有望成为"），不能当事实直接搬——去原文块/链接核实硬事实。
> ⚠️ 第 4 种带原文：原文就在眼前，洗稿更"顺手"——务必"读原文→记事实笔记→关原文→凭笔记写"，深写逐句跑 Q1；英文原文不许逐句直译当正文。

## Step 2 · 选题评分 + 组稿（不替用户拍板）

完整模型见 `references/topic-selection.md`。要点：

- 按主题聚类（如：大模型/融资、硬件、政策监管、安全、终端产品……）。
- **三道门槛**（版权 G1 / 事实 G2 / 定位 G3）一票否决，不过槛的降级或弃用。
- 过槛条目按**三维读者价值评分**（相关度 ×0.4 + 话题度 ×0.3 + 增量 ×0.3，再乘时效系数）排序——
  **不沿用上游卡片 ⭐️ 评分**（⭐️ 是新闻重要度，不等于公众号传播价值，仅作初筛线索）。
- **少而精（默认）**：只精选**总分最高的 2-3 条深写**，其余不收（不进简讯、不留尾）。
  **输入已是预选的 2-3 条**（常带原文）则**不再筛、全做深写**。
- **组稿**：定封面条目、检测主线（有主线则用 1-2 句导语点题，不用速览）。
- 把"精选哪 2-3 条 + 主线 + 其余未选一句话"**交用户确认**，再进入成稿。除非用户说"你来定"。

## Step 3 · 逐条事实提取 → 事实卡片

对每条入选资讯，**优先读输入里的 `> [!quote]- 原文` 块**（无原文且需核实时才联网），关掉原文，凭笔记填写事实卡片：

```
【资讯名称】
【一句话摘要】          ← 自己的话，不照抄原标题/导语
【发生了什么】          ← 核心事件，只写已证实的
【关键信息】            ← 主体 / 动作 / 时间 / 地点
【涉及数据】            ← 金额、比例、用户数等；逐项标来源状态（已证实 / 待核实）
【涉及主体】            ← 公司、人物、机构（用准确全称）
【来源类型】            ← 按 S/A/B/C 分级（见 copyright-guardrails.md 9.2）：S 一手权威 / A 多源权威 / B 单源独家 / C 二手转述
【可信度评分】          ← 高 / 中 / 低（见下表）
【版权风险评分】        ← 低 / 中 / 高（见 copyright-guardrails.md）
【是否适合发布】        ← 适合 / 降级处理 / 不建议
【需要谨慎表达的地方】  ← 哪些话必须保留不确定性
```

**可信度评分参考**：
- 高：公司公告、政府/监管原文、≥2 家权威媒体一致。
- 中：单一权威媒体、或权威媒体引述但未被官方确认。
- 低：单一自媒体、"据悉/知情人士/市场传闻"、未经证实。

## Step 4 · 版权 + 事实风险判断

逐条套用 `copyright-guardrails.md` 和 `fact-risk-control.md`：
- 版权：是否有照搬标题/开头/结构的风险？来源是不是另一个公众号（需回溯一手源）？有没有用原图？
- 事实：哪些是事实、哪些是推测、哪些是观点？高风险领域（融资/股价/政策/医疗/法律）措辞是否降级？

## Step 5 · 成稿 + 排版

- **封面总标题**：先产出**主推** 1 个（围绕封面条目/主线，见 `titles-and-style.md`；备选可附）。
- **L1 正文（少而精，默认）**：按 `digest-templates.md` 少而精模式写——**1-2 句导语点主线 + 2-3 条深写**（每条 600–900 字），
  **不用速览、不用简讯**；条目用**连贯叙事**（轻标记 + 叙事主体），中间把机制/背景/影响/名词**就地讲透**、通俗但克制；"仍需注意"只在有不确定性时写。
  - 版权红线（D16 + v3.1）：深写的"丰富"来自**自己再解释**（大白话重讲、补背景、打比方），**不是把原文展开复述/直译**；
    **每段逐句跑量化回查（与原文无连续 ≥13 字重合）**——原文在输入里时尤其要严。
- **L2 排版稿**：成稿后按 `wechat-formatting.md` 转成可直接粘贴的微信内联样式稿
  （模型只产 markdown，HTML 由确定性转换生成），并附"粘贴后自查 3 点"。
- **不达标条目的降级规则**（已与用户确认）：版权或来源不达标（C 级单一自媒体源 / 纯传闻）的，
  不强行写长，降级为"一句话事实 + 标注（待核实，建议补充来源）"，放进简讯区做最小处理。
- 措辞套用 `titles-and-style.md` 的白名单（含口语类）/黑名单。

## Step 6 · 发布前检查 + 变现建议

- 跑 `prepublish-checklist.md`，输出 ✅/⚠️/❌ 清单；**版权量化回查 Q1–Q4 或 AI 声明出现 ❌ 即不建议发布**，给修改建议。
- 给 `monetization.md` 的轻量变现建议（说明是否适合插广告、是否会影响可信度）。

## 最终输出结构（默认）

```
【1. 选题清单与组稿方案】       ← 评分明细、封面条目、主线、落选理由与去向
【2. 封面总标题 + 条目标题】     ← 封面总标题(主推 1 个，可附备选)；各条小标题
【3. 公众号正文(L1)】           ← 导语点主线 + 2-3 条深写(每条 600–900 字) markdown
【4. 微信排版稿(L2)】           ← 内联样式，可直接粘贴；附粘贴后自查 3 点
【5. 来源与风险提示】           ← 哪些需以官方披露为准、哪些不可夸大
【6. 版权与合规检查】           ← 量化回查 Q1–Q4 结果 + AI 声明确认 + 未打原创确认
【7. 配图建议(文字版)】         ← 绝不用原文图
【8. 变现建议】                 ← 低门槛、不污染可信度
```
