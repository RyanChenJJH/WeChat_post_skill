# 事实安全与风险控制细则

## 事实 / 推测 / 观点 三分法

| 类型 | 判断标准 | 推荐写法 |
|---|---|---|
| 事实 | 有明确来源、时间、主体、数据 | 据公告显示 / 据公开资料显示 / 该公司表示 |
| 推测 | 基于事实延展，但未被证实 | 这可能意味着 / 或将影响 / 有望带来 |
| 观点 | 作者的判断、分析 | 目前来看 / 一个值得关注的信号是 / 从公开信息看 |

> 本 skill 定位"轻度整理"，观点要克制：能不下判断就不下，能用"值得继续观察"就不用"必将"。

## 可信度与来源分级的对应

可信度判断直接对应 `copyright-guardrails.md` 9.2 的来源分级 S/A/B/C（事实卡片【来源类型】填的就是这个级别）：

| 来源级 | 典型 | 可信度 |
|---|---|---|
| S 一手权威 | 政府/监管公告、法院文书、上市公司公告、公司官方、官方文档 | 高 |
| A 多源权威 | ≥2 家权威媒体交叉一致 | 高 |
| B 单源独家 | 单一权威媒体独家/专访/付费墙 | 中（未被官方/第二源确认时） |
| C 二手转述 | 自媒体公众号、Telegram、聚合号、传闻 | 低（仅作线索，须回溯一手源） |

## 不确定信息处理

| 原始表达 | 处理规则 |
|---|---|
| 据悉 / 有消息称 | 保留不确定性，不写成已证实事实；建议补充来源 |
| 知情人士称 | 标注为未公开确认，不作为核心结论 |
| 公开资料显示 | 可用，但不扩展原文没给出的结论 |
| 市场传闻 / 传闻 | 不作为核心事实；要么标"传闻、待证实"，要么不收录 |
| 业内认为 | 说明这是观点集合，不写成事实 |

## 高风险领域规则

| 领域 | 风险 | 处理规则 |
|---|---|---|
| 投融资 | 易变成投资建议 | 可陈述融资事实，不预测收益、不建议买卖、不断言"将成龙头/上市" |
| 股价 / 市值 | 时效性强 | 必须标注时间点，不预测涨跌 |
| 政策 / 监管 | 易误读 | 建议引用/链接原文，用原文口径，不替代法律解释 |
| 医疗 / 健康 | 关系人身 | 只做信息整理，不做诊疗、疗效、用药结论 |
| 法律 / 司法 | 易定性不当 | 用"涉嫌""争议""有待确认"；判决类以官方通报为准 |
| 金融投资 | 合规风险高 | 禁用"稳赚""必涨""抄底""无风险" |

## 高风险表达替换（精简版，完整白/黑名单见 titles-and-style.md）

| 高风险 | 安全替换 |
|---|---|
| 一定会 / 必将 | 可能会 / 有望 |
| 彻底改变 / 颠覆行业 | 正在影响 / 推动行业变化 |
| 证明了 | 释放出一个信号 |
| 全球第一 / 最强 | 在公开信息中表现靠前 / 较具竞争力 |
| 暴涨 | 明显上涨 / 阶段性上涨 |
| 稳赚 / 保证收益 | 存在不确定性，需结合实际判断 |

## 必须提醒用户补充来源的情况

- 涉及金额、排名、市场份额、增长率，但没有一手出处。
- 涉及政策、监管、法律解释，或医疗、投资、股价、收益预测。
- 原文只有观点没有事实。
- 用户要求使用"第一/唯一/必然/稳赚"等强结论。
- 内容会影响企业声誉、公众判断或商业决策。

## 建议核对的情况（优先用输入原文，联网为可选）

> v3.1：若输入已带原文（`> [!quote]- 原文` 块），**优先用原文核对**，不必联网。以下"联网"仅在**输入无原文且需核实**、且联网可用时才走；联网不可用就按断网降级（仅用已有摘要硬事实，未证实标待核实）。

遇到以下情况，核对一手源（输入原文优先，否则联网）后再写：
- 关键金额、轮次、投资方名称（融资类）。
- 政策/法规的具体条款、判决的具体罪名和刑期。
- 最新股价、市值、榜单数据。
- 任何"据悉/传闻/知情人士"且打算保留在正文里的信息。

> 注：未来若部署到无联网能力的第三方 API 网页端，本节降级为"只用已抓取的事实卡片，
> 凡卡片未证实的一律标注待核实"。
