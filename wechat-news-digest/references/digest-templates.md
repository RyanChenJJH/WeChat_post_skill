# 输出模板与示例

## 模板 A · 少而精深写（默认）

**默认只精选 2-3 条，每条深写讲透**（不要速览、不要简讯、不要"其他快讯"尾）。要的是质不是量。
条目用**连贯叙事**（不是拆成一堆分点），而叙事的"丰富"来自**你自己的再解释**，绝不是把原文展开复述
（见 `copyright-guardrails.md` 的"丰富叙事核心原则"与"原文在手时的深写纪律"）。
**原文就在输入里时，洗稿风险更高——深写每段都要逐句跑 Q1（与原文无连续 ≥13 字重合）。**

整体结构（≤3 条模式）：

```
[封面总标题]（主推 1 个，见 titles-and-style.md；备选可附）

[导语]（1–2 句，点出本期主线 / 为什么这 2-3 条值得花时间。不用"本期速览"列表）

[条目 1：深写 600–900 字]
[条目 2：深写 600–900 字]
([条目 3：深写 600–900 字])

[结尾区]：克制收束 + 互动引导 + AI 声明 + 参考来源
```

### 深写条目骨架（轻标记 + 连贯叙事，中间是重点）

```
[条目小标题]

发生了什么：<1–2 句硬事实，点明 主体/动作/时间>

<连贯叙事主体——这是深写的核心，约 400–700 字，分 2–4 个自然段，把"中间部分"讲透。
 按需覆盖（不必每项都写，挑这条最该讲的）：
 · 来龙去脉：这事怎么发生的、此前的背景/时间线；
 · 机制/原理：它到底怎么做到的——用大白话 + 打比方讲清楚（原"名词卡"在此展开）；
 · 关键细节与数字：织进句子讲，不罗列；数字以原文/官方为准；
 · 影响/意义：对读者、对行业意味着什么（轻度、克制）；
 · 张力/争议/限制：如有不同声音或不确定，如实带出（尤其原文里写的 limitation）。
 全程通俗但克制；版权：靠自己把事讲明白，不照搬/不扩写原文表达。>

为什么值得关注：<1–2 句收束本条对读者的意义>
（仍需注意：<条件字段，仅在有待核实数据/传闻/高风险时才写>）
来源：<媒体名 · 日期>
```

写"够"的判断：读者读完能**说清这事是什么、怎么回事、与我何干**，并学到一点新东西；但不灌水、不超 900 字。
要点：
- **"关键信息""名词卡""背景卡"不单列**，全部融进叙事主体——就地讲清楚。
- 2-3 条都是"深写"，不再分头条/标准/简讯层级。
- "仍需注意"是条件字段，不是每条都挂。

> **备选 · 分层节奏（多条大合集）**：仅当用户明确要"做一期多条大合集"时启用——
> 那时才用「封面总标题 + 钩子式本期速览(每条≤15字) + 头条区(深写) + 标准区(精简) + 简讯区(一句话) + 结尾区」。
> 默认不走这个，走上面的少而精。

### 通俗但克制（口吻边界）

- 鼓励：说白了 / 打个比方 / 换句话说 / 这意味着 / 你可以理解为。
- 禁止：颠覆 / 必将 / 全网第一 / 稳赚 / 重大利好 / 炸裂 / 不容错过。
- 严肃政策/法律/医疗/司法条目：可通俗解释，但收着写、不调侃。
- 自检一句：通俗是为了"读懂"，不是为了"耸动"。

## 模板 B · 单条资讯轻解读（800–1500 字）

封面总标题 + 导语 + 同样的叙事骨架展开（发生了什么 → 连贯叙事主体，把背景、术语、影响讲透 →
为什么值得关注 →（仍需注意）→ 来源 + AI 声明）。比合集里的头条更长、更展开，但纪律一样：丰富靠自己再解释。

## 模板 C · AI 工具 / 公司动态简报

同类资讯聚成专题，每条用标准区叙事骨架，前面加一句专题导语，结尾区带 AI 声明与参考来源。

## 走通的示例（用本项目 6/9 速递的真实条目）

### 示例 1：头条区叙事式深写（封面条目，含融入式名词/背景解释）

**原始卡片**：微信开放平台宣布小程序可接入微信 AI 生态，分自动/开发两种模式，目前内测。

**事实提取（关原文后凭笔记）**：主体=微信开放平台；动作=宣布小程序可接入 AI 生态、可被 AI 推荐调用；
两种模式（自动：授权读源码、无需额外开发；开发：自主开发、需评测审核）；状态=内测。

**成稿（头条区一条，叙事式）**：

```
01｜微信小程序要能被 AI 调用了，开放平台开了两种接入模式

发生了什么：微信开放平台宣布，小程序可以接入微信的 AI 生态，未来有机会被微信 AI 直接推荐、调用，目前还在内测。

接入分两种玩法，还能同时开：一种叫"自动模式"，开发者几乎不用额外干活，授权后平台在提审时读一下源码，
AI 就能直接操作页面；另一种是"开发模式"，自己做个性化定制，但得过平台的评测和审核。说白了，微信是想把
那几百万个小程序，从"你自己点开找"变成"你说一句、AI 帮你调"。打个比方，以后想点杯咖啡，可能不用再翻
小程序，直接跟微信 AI 说一声就行。这背后是腾讯憋的一步棋：把社交关系、公众号视频号，再加上海量小程序，
攒成一个别人抄不走的"AI 入口"。

为什么值得关注：微信几乎人人在用。把 AI 装进这个国民级入口，意味着"用一句话指挥服务"这件事，离普通人又近了一步。

仍需注意：功能还在内测，开放范围、时间表、实际效果都以微信开放平台的正式说明为准。

来源：微信开放平台 · 6/8
```

> 看这条怎么同时做到"丰富"和"不洗稿"：术语（自动/开发模式）用**自己的大白话**重讲，打比方（点咖啡）、
> 背景（腾讯的入口战略）都是**自己组织的再解释**，没有照搬原文的句子和结构——这就是 D16 原则。

### 示例 2：标准区叙事式精简（高可信度条目 + 融入背景）

```
02｜月之暗面新一轮融资落地，估值升到百亿美元量级

发生了什么：据公开资料，AI 公司月之暗面完成新一轮融资，本轮超 7 亿美元，由阿里、腾讯等领投，估值升至 100 亿美元量级。

把时间线拉一下你会更有感觉：这家公司 2023 年才成立，靠"超长上下文"的 Kimi 出圈——简单说就是一次能读进
很长的文档不容易忘前面，常被归到国内"AI 四小龙"。这一轮累计融资已超 12 亿美元，旗下的 Kimi K2.5 还作为
开源多模态模型上线了 OpenRouter（开源＝模型公开可自由部署，多模态＝能同时处理文字和图像）。

为什么值得关注：在国内大模型公司里，这一轮的估值和融资节奏都算靠前，说明资本对头部团队还在持续下注。

仍需注意：有消息称 Kimi 近期收入、海外占比涨得快，但这类数字属于解读性信息，以公司官方披露为准，本文不据此推断后续走势。

来源：Telegram·zaihuapd · 6/8（融资细节待一手公告确认）
```

> 注意风险处理：估值/融资是事实（注明来源）；"20 天收入超全年""海外超国内"降级为"有消息称…以官方为准"，
> 不写成既定事实、不下"将成中国 OpenAI"判断。名词（开源/多模态）就地用括注讲白，不抄原文解释。

### 示例 3：降级处理 → 简讯区（不达标条目）

```
· 另有传闻称 iOS 27 可能维持现有机型兼容范围（最低 iPhone 11）。该说法暂无官方来源，仅供参考。
```

### 示例 4：高风险领域（政策/司法）的叙事——通俗但收着

```
03｜青岛一男子暗记助记词，盗走 107 枚比特币获刑十年九个月

发生了什么：据司法机关通报，青岛一名男子在帮人设置数字钱包时，暗中记下部分助记词，之后盗走 107 枚比特币变现。

这里有个对普通人也有用的知识点：助记词就是加密钱包的"万能钥匙"——一串按固定词库生成的单词，谁拿到它谁就能
转走里面的钱，哪怕只泄露一部分，破解难度也会大降。据通报，这名男子正是记下部分单词后暴力破解了其余部分，
变现 66 万余元；法院按实际变现金额、而不是比特币当时市价来认定盗窃数额，最终判了十年九个月。

为什么值得关注：与其说是个案，不如说是提醒——助记词的物理安全和密码安全一样重要，别让任何人有机会看到、记下它。

仍需注意：案情细节与定性以司法机关官方通报为准；本文不对加密资产的法律属性或投资价值作判断。

来源：最高人民检察院/司法机关通报 · 6/7
```

> 政策/司法类可以通俗（讲清"助记词是什么"），但措辞收着：用"据通报""法院以…判处…"客观陈述，不调侃、不演绎。
