# 微信排版稿（L2）

成稿后，除纯文本 markdown（L1）外，再产出一份**可直接粘贴进公众号编辑器的排版稿（L2）**。

## 生成方式（重要工程判断）

**模型只产 L1 markdown；L2 的 HTML 由"确定性转换"产生，不是模型自由发挥写出来的。**

- 原因：排版是确定性任务。让大模型直接写 HTML 又贵、又不稳、还难审；而 md→微信 HTML 是固定映射，
  交给工具/代码又快又稳。
- Phase 1（Claude/Codex 端）：调用环境内现成的 md→微信 HTML 转换工具链生成；
  没有工具时，按下方 token 表做**机械映射**（仍是固定规则，不要即兴改结构）。
- Phase 2（自建网页 + API）：在应用层实现一个 md→微信 HTML 转换器，不耗模型 tokens。

## 微信编辑器三条硬约束（不遵守粘贴即坏）

1. **全部样式内联**：用 `style="..."` 写在标签上。编辑器会剥离外部 CSS、`class`、`<style>`、
   JS、`iframe`、`position` 定位——这些一律不要用。
2. **正文外链不可点**：微信正文里非微信生态的链接点不动。处理办法：
   - 正文中的来源显示为**纯文本**「媒体名 + 日期」（如：新华社 · 6/8），不放裸链接；
   - 完整链接集中到结尾区「参考来源」列表；
   - 最重要的一条可建议设为"阅读原文"。
3. **图片只放占位说明**：配图仍走文字版配图建议（不嵌原文图）。HTML 里用占位文字标明"此处插图：……"。

## 默认样式 token（主色取自 account-profile.md，默认 `#07C160`）

| 元素 | 样式 |
|---|---|
| 正文段落 | `font-size:15px; line-height:1.75; margin:0 0 16px;` |
| 头条区标题 | `font-size:17px; font-weight:bold;` + 主色左边框或下划线 |
| 区块标题（栏目名） | `font-size:16px; font-weight:bold; color:主色;` |
| 简讯区 | `font-size:14px; color:#888;` |
| 来源行 / AI 声明行 | `font-size:13px; color:#999;` |

## 粘贴后自查 3 点（随 L2 一并输出给用户）

1. 样式是否保留（标题/区块色/字号正常，没有被编辑器吃掉）。
2. 有无乱码或错位。
3. 来源是否按"正文纯文本 + 文末参考列表"呈现，没有残留不可点裸链。
