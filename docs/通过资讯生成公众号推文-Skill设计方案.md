# 通过资讯生成公众号推文 Skill 设计方案

> 版本：v1.0  
> 适用场景：将资讯、新闻、公告、行业动态、公开资料分析后，生成适合微信公众号发布的推文内容。  
> 设计定位：资讯分析 + 选题策划 + 微信公众号推文生成 + 事实风险审校。  
> 默认策略：目标读者默认为行业从业者；写作用途默认为专业解读；写作风格默认为专业理性；字数默认 1000-1500 字，并允许 AI 根据信息密度自动调整。

## 一、Skill 的核心定位

### 1.1 能力边界

这个 Skill 不是简单的“改写器”，而是一个面向微信公众号内容生产的结构化写作工作流。它应先完成资讯拆解、价值判断、文章类型选择和风险识别，再生成标题、大纲、正文和发布辅助内容。

核心能力包括：

- 从原始资讯中提取事实、观点、数据、主体、事件和影响对象。
- 判断资讯适合写成哪类公众号文章。
- 根据目标读者和账号定位选择文章角度。
- 生成多风格标题、文章大纲、正文初稿、金句、配图建议和朋友圈转发文案。
- 对事实、数据、政策、投融资、医疗、法律、股价、市值等高风险内容进行表达降级和风险提示。
- 在信息不足时追问用户；在用户要求直接写时使用默认配置并标注假设。

### 1.2 适合处理的资讯类型

| 类型 | 适用说明 | 推荐文章方向 |
|---|---|---|
| 科技资讯 | AI、互联网、硬件、软件、SaaS、开源、产品更新 | 科技媒体稿、新闻解读、科普解释 |
| 财经资讯 | 财报、市场变化、商业模式、产业链变化 | 财经科技分析、行业分析 |
| 企业动态 | 战略发布、组织调整、签约合作、业务升级 | 新闻解读、品牌宣传、商业案例 |
| 政策信息 | 政策发布、监管动态、行业规范 | 政策解读、行业影响分析 |
| 行业新闻 | 行业趋势、竞争格局、供需变化 | 行业分析、趋势判断 |
| 产品发布 | 新品上线、功能升级、解决方案发布 | 产品解读、品牌宣传、科普解释 |
| 投融资事件 | 融资公告、并购、上市、估值变化 | 投融资解读、赛道分析 |
| 人物访谈 | 创始人观点、专家访谈、公开演讲 | 人物故事、观点评论 |
| 品牌营销信息 | 活动、案例、品牌理念、用户故事 | 品牌宣传、商业案例、营销转化 |
| 热点事件 | 平台变化、舆论事件、行业热点 | 热点借势、观点评论、新闻解读 |

### 1.3 不适合直接处理的内容

| 内容类型 | 风险 | 推荐处理方式 |
|---|---|---|
| 信息严重不足 | 容易编造背景和结论 | 先追问来源、背景、目标读者 |
| 来源不明 | 可信度不足 | 标记低可信，不写强结论 |
| 医疗、法律、投资结论 | 高合规风险 | 只做信息整理，不提供建议 |
| 强事实核查内容但未给出处 | 可能误导读者 | 提醒补充来源或建议联网核查 |
| 过度营销、虚假宣传 | 损害账号可信度 | 降级表达，删除绝对化承诺 |
| 涉及敏感人物、监管、股价、市值 | 表达风险高 | 使用谨慎措辞并提示核查 |
| 用户要求编造数据、案例、引用 | 事实风险高 | 明确拒绝编造，改为“可补充数据位” |

### 1.4 最终产出

默认输出以下内容：

1. 资讯摘要
2. 核心观点
3. 推荐文章类型
4. 选题角度
5. 标题备选
6. 文章大纲
7. 正文初稿
8. 金句提炼
9. 配图建议
10. 朋友圈转发文案
11. 风险提示
12. 可优化方向

如果用户明确要求“先给策划方案，不写正文”，则只输出资讯摘要、文章类型、选题角度、标题备选、文章大纲和风险提示。

## 二、输入信息设计

### 2.1 必填信息

| 字段 | 说明 | 默认处理 |
|---|---|---|
| 原始资讯内容 | 新闻、公告、采访、资料、链接摘要、会议纪要等 | 必须提供 |
| 目标读者 | 普通用户、行业从业者、企业客户、投资人等 | 默认：行业从业者 |
| 文章用途 | 快讯、解读、宣传、评论、科普等 | 默认：专业解读 |
| 文章风格 | 专业理性、深度分析、企业官方等 | 默认：专业理性 |
| 字数范围 | 600-900、1000-1500、1800-2500 或自定义 | 默认：1000-1500，允许 AI 根据信息密度调整 |

### 2.2 可选信息

| 字段 | 说明 | 影响 |
|---|---|---|
| 账号定位 | 科技号、财经号、企业号、行业号、个人 IP 号 | 决定语言、角度和标题尺度 |
| 是否标题更吸引人 | 可选择稳重、适中、爆款 | 影响标题表达强度 |
| 是否需要专业分析 | 是否加入行业背景、数据、趋势 | 影响文章深度 |
| 是否需要引用数据 | 是否保留或扩展数据分析 | 仅在有来源时引用 |
| 是否加入品牌观点 | 是否植入企业立场或品牌价值 | 适合企业号、品牌号 |
| 是否生成多个版本 | 专业版、传播版、企业版、短版 | 适合 A/B 测试 |
| 是否联网核查 | 是否需要验证公开来源 | 默认不联网，高风险时建议核查 |
| 是否需要配图建议 | 是否输出封面图、正文图建议 | 默认输出建议 |
| 是否需要朋友圈文案 | 是否输出转发文案 | 默认输出 1-3 条 |

### 2.3 AI 自动判断的信息

| 自动判断项 | 判断依据 | 输出 |
|---|---|---|
| 资讯类型 | 主题、主体、关键词、事件性质 | 科技、财经、政策、企业动态等 |
| 信息密度 | 是否包含时间、主体、数据、背景、影响 | 高、中、低 |
| 传播价值 | 新鲜度、冲突点、趋势性、读者相关性 | 高、中、低 |
| 风险等级 | 是否涉及金融、政策、医疗、法律、股价等 | 低、中、高 |
| 推荐文章角度 | 读者利益、行业影响、趋势信号、品牌价值 | 3-5 个角度 |
| 推荐文章类型 | 信息密度、传播目标、资讯类型 | 新闻解读、行业分析等 |
| 是否需要补充来源 | 数据缺口、来源缺口、结论强度 | 补充问题或风险提示 |
| 是否适合深度稿 | 背景和因果链是否充分 | 深度稿、快讯稿、评论稿或营销稿 |

## 三、资讯分析模块

### 3.1 模块目标

资讯拆解模块的目标是先理解资讯，不急着写正文。AI 必须先判断原始信息里哪些是事实，哪些是观点，哪些数据值得保留，哪些表述存在风险，以及这条资讯为什么值得目标读者关注。

### 3.2 分析流程

1. 提取基础信息：时间、地点、主体、事件、动作、结果。
2. 提取事实信息：已经明确发生、可被验证的信息。
3. 提取观点信息：原文中的判断、预测、评价和立场。
4. 提取关键数据：金额、比例、增长率、排名、用户数、时间节点。
5. 判断来源可信度：是否来自公告、权威媒体、公开资料、单一信源或传闻。
6. 判断传播价值：新鲜度、反差性、影响面、读者相关性。
7. 判断影响对象：用户、企业、行业、投资方、监管方、上下游。
8. 判断风险点：夸大、不确定、敏感、误导、缺来源。
9. 生成选题角度：至少 3 个，并按推荐度排序。
10. 生成写作禁区：明确不能夸大、不能编造、不能下强结论的部分。

### 3.3 输出格式

```markdown

- 可延展观点：
- 不应直接当作事实的判断：

### 5. 值得保留的数据
- 数据：
- 来源状态：
- 可用方式：

### 6. 风险表述
- 高风险表达：
- 替换建议：
```

## 四、文章类型判断模块

### 4.1 类型选择原则

AI 应根据资讯内容、信息密度、读者需求和风险等级，选择最适合的文章类型。默认机制是自动判断，默认倾向是“专业解读”。

### 4.2 文章类型表

| 类型 | 适合资讯 | 文章结构 | 写作重点 | 需要避免 |
|---|---|---|---|---|
| 新闻解读型 | 新政策、新产品、新事件、企业公告 | 事件 -> 背景 -> 原因 -> 影响 -> 结论 | 讲清发生了什么、为什么重要 | 不要只复述新闻 |
| 行业分析型 | 行业趋势、市场变化、竞争格局 | 现象 -> 结构 -> 变量 -> 趋势 -> 机会 | 提炼行业逻辑和趋势信号 | 避免空泛大词 |
| 观点评论型 | 有争议、有态度、有讨论度的事件 | 观点 -> 论据 -> 反方 -> 判断 -> 启发 | 观点清晰、有边界 | 避免情绪过重和定性过猛 |
| 科普解释型 | 新技术、新概念、新规则 | 问题 -> 解释 -> 例子 -> 应用 -> 总结 | 把复杂问题讲简单 | 避免术语堆砌 |
| 商业案例型 | 企业动作、增长案例、组织方法 | 案例 -> 做法 -> 原因 -> 启发 -> 复盘 | 提炼可借鉴经验 | 避免神化企业 |
| 品牌宣传型 | 企业发布、活动、产品升级、客户案例 | 痛点 -> 行动 -> 价值 -> 案例 -> 号召 | 传递品牌观点和用户价值 | 避免硬广感太强 |
| 投融资解读型 | 融资、并购、上市、估值变化 | 事件 -> 金额 -> 投资方 -> 赛道 -> 信号 | 看懂资本为何关注 | 不做投资建议 |
| 人物故事型 | 访谈、演讲、创始人故事 | 人物 -> 转折 -> 选择 -> 观点 -> 价值 | 写出人物和时代关系 | 避免过度包装 |
| 热点借势型 | 热点话题、平台变化、舆论事件 | 热点 -> 关联 -> 洞察 -> 启发 -> 互动 | 借热点表达自身观点 | 避免蹭热点失当 |
| 快讯简报型 | 信息明确但材料较少的短资讯 | 一句话 -> 三要点 -> 影响 -> 后续 | 快、准、短 | 避免强行深度 |

### 4.3 默认判断规则

- 信息完整、有背景、有影响：优先新闻解读型。
- 信息涉及行业变化：优先行业分析型。
- 信息很短但重要：优先快讯简报型。
- 企业自有内容：优先品牌宣传型。
- 涉及融资、并购、上市：优先投融资解读型，并开启风险审校。
- 用户未说明时：默认“新闻解读型 + 专业理性风格”。

## 五、标题生成模块

### 5.1 标题总原则

标题要有点击理由，但不得虚假承诺、绝对化判断、低俗化表达、过度夸大，也不能与正文内容不匹配。

### 5.2 标题类型、公式与示例

| 标题类型 | 标题公式 | 示例 | 适用场景 |
|---|---|---|---|
| 稳重专业型 | 主体 + 动作 + 行业意义 | 某公司发布新一代 AI 平台，企业级应用进入新阶段 | 企业号、行业号、专业解读 |
| 自媒体爆款型 | 事件 + 反差/冲突 + 读者利益 | 这次 AI 产品更新，真正值得关注的不是功能 | 个人号、内容号、热点稿 |
| 财经科技分析型 | 事件 + 资本/技术/市场信号 | 从这轮融资看，AI 应用层正在出现新分化 | 财经、科技、投融资分析 |
| 情绪共鸣型 | 人群处境 + 事件启发 | 越来越多中小企业，正在被这类工具重新改写工作方式 | 大众传播、职场类内容 |
| 悬念反差型 | 表面现象 + 深层变化 | 看似一次产品更新，背后可能是行业竞争逻辑变了 | 热点解读、趋势稿 |
| 趋势判断型 | 趋势 + 变化方向 | AI 办公工具进入下半场：从效率工具走向业务系统 | 行业趋势、深度分析 |
| 品牌官方型 | 品牌动作 + 价值表达 | 某品牌升级智能解决方案，助力企业提升运营效率 | 企业发布、品牌宣传 |

### 5.3 标题安全规则

| 应避免表达 | 推荐替换 |
|---|---|
| 彻底颠覆 | 正在改变 |
| 必将取代 | 可能影响 |
| 全球第一 | 在相关领域表现突出 |
| 一夜爆火 | 近期受到关注 |
| 暴涨 | 明显上涨 / 阶段性上涨 |
| 唯一答案 | 一种可参考路径 |
| 全网最强 | 较具竞争力 |
| 保证有效 | 有助于提升 / 可能改善 |

## 六、文章结构模块

### 6.1 标准结构

一篇完整的公众号文章默认包含三层结构：

1. 开头：用一句话抓住读者，交代资讯背景，引出核心观点。
2. 正文：资讯复述、背景解释、原因分析、影响分析、趋势判断、读者启发。
3. 结尾：总结观点，升华主题，留下金句，引导评论、点赞、转发。

### 6.2 模板一：新闻解读结构

适合资讯：新闻、公告、政策、产品发布、企业动态。  
适合语气：专业、清楚、克制。

结构顺序：

1. 导语：说明这件事为什么值得关注。
2. 发生了什么：复述资讯核心事实。
3. 为什么现在发生：补充背景和原因。
4. 它影响了谁：分析用户、企业、行业影响。
5. 背后释放什么信号：提炼趋势和行业意义。
6. 结尾：总结观点并引导互动。

示例小标题：

- 这次发布，核心不只是一个新功能
- 真正值得关注的是它背后的行业信号
- 接下来，影响可能会出现在三个层面

### 6.3 模板二：行业分析结构

适合资讯：产业变化、赛道竞争、市场变化、长期趋势。  
适合语气：分析型、理性、略有洞察感。

结构顺序：

1. 导语：指出一个行业变化正在出现。
2. 表面是事件，本质是结构变化。
3. 行业过去靠什么增长。
4. 当前关键变量发生了什么变化。
5. 哪些公司或人群会受到影响。
6. 未来可能出现的三种走向。

示例小标题：

- 表面是一次融资，本质是赛道进入新阶段
- 行业竞争的核心变量正在变化
- 真正的机会，可能不在流量端

### 6.4 模板三：观点评论结构

适合资讯：有争议、有公共情绪、有观点空间的事件。  
适合语气：有判断、有锋芒，但不过激。

结构顺序：

1. 导语：先抛出明确判断。
2. 这件事不能只看表面。
3. 支持判断的三个理由。
4. 反方观点也有合理之处。
5. 真正的问题是什么。
6. 读者能从中得到什么启发。

示例小标题：

- 这不是一个简单的是非题
- 真正值得讨论的，是背后的规则变化
- 争议之外，我们更应该看到这个信号

### 6.5 模板四：品牌宣传结构

适合资讯：企业动态、产品发布、案例传播、品牌活动。  
适合语气：官方但不僵硬，强调价值而非硬推销。

结构顺序：

1. 导语：从用户痛点或行业背景切入。
2. 行业正在面临什么问题。
3. 品牌做了什么。
4. 这件事解决了什么价值。
5. 真实场景或案例如何体现。
6. 品牌观点与未来计划。

示例小标题：

- 从一个真实需求开始
- 产品升级背后，是对用户场景的重新理解
- 让技术真正进入业务现场

### 6.6 模板五：热点事件结构

适合资讯：热点话题、平台变化、舆论事件、突发行业热点。  
适合语气：快、准、有洞察，避免情绪过载。

结构顺序：

1. 导语：热点发生了什么。
2. 事件本身并不复杂。
3. 为什么它会引发关注。
4. 它折射出什么更大的问题。
5. 不同人群该怎么看。
6. 热度过去后，真正留下的是什么。

示例小标题：

- 热点之外，更大的变化已经出现
- 为什么这件事会让这么多人关注
- 真正留下来的，不只是一次讨论

## 七、写作风格模块

| 风格 | 语言特征 | 句式特征 | 适合场景 | 不适合场景 | 示例句子 |
|---|---|---|---|---|---|
| 专业理性 | 清楚、克制、少情绪 | 中短句，逻辑连接明确 | 默认通用、行业解读 | 强娱乐热点 | 这件事的重点，不在于单一产品更新，而在于行业应用边界正在扩大。 |
| 深度分析 | 强因果链、背景充分 | 长短句结合，层层递进 | 行业稿、财经稿、趋势稿 | 快讯短稿 | 如果把这次变化放回产业链中看，它更像是供需关系变化后的必然结果。 |
| 犀利评论 | 判断明确、有锋芒 | 短句多，转折强 | 争议事件、观点号 | 企业官方、政策稿 | 真正的问题不是谁先入局，而是谁能把场景做深。 |
| 克制高级 | 简洁、有留白、少口号 | 短句和抽象表达结合 | 品牌号、高端号 | 大众科普、快讯 | 变化不是突然发生的。只是这一次，它被更多人看见了。 |
| 口语化自媒体 | 易懂、有节奏、有代入 | 多用设问、转折、解释句 | 大众传播、个人号 | 严肃政策、法律医疗 | 说白了，这不是一次普通更新，而是一次方向选择。 |
| 企业官方 | 稳健、价值导向、少情绪 | 规范句、并列句较多 | 品牌宣传、企业发布 | 热点评论、吐槽稿 | 未来，公司将持续围绕用户需求，推动产品和服务升级。 |
| 财经媒体 | 数据、市场、资本、赛道 | 信息密集，逻辑偏硬 | 投融资、财报、商业分析 | 情绪故事 | 从资本流向看，市场对该赛道的长期价值仍保持关注。 |
| 科技媒体 | 技术、产品、场景、效率 | 概念解释 + 场景落地 | AI、硬件、SaaS、开源 | 纯情绪稿 | 这次更新的关键，在于模型能力与业务流程的进一步结合。 |
| 故事化叙述 | 人物、冲突、转折、细节 | 场景句、动作句较多 | 人物访谈、创业故事 | 政策快讯 | 三年前，他还在解释这个方向为什么重要。现在，市场已经开始给出答案。 |
| 营销转化型 | 痛点、价值、行动 | 问题句、利益句、行动句 | 企业服务、产品推广 | 公共新闻、严肃政策 | 如果你的团队也在面对类似效率问题，这类方案值得提前关注。 |

## 八、事实安全与风险控制模块

### 8.1 基本原则

AI 必须区分事实、推测和观点，不得把未经证实的信息写成确定事实，不得编造数据、来源、人物评价、行业排名或案例。

| 类型 | 判断标准 | 推荐写法 |
|---|---|---|
| 事实 | 有明确来源、时间、主体、数据 | 公告显示 / 公开资料显示 / 该公司表示 |
| 推测 | 基于事实延展，但未被证实 | 这可能意味着 / 有望带来 / 或将影响 |
| 观点 | 作者判断、评论、分析 | 从行业角度看 / 一个值得关注的信号是 |

### 8.2 不确定信息处理

| 原始表达 | 处理规则 |
|---|---|
| 据悉 | 保留不确定性，不写成已证实事实 |
| 知情人士称 | 标注为未公开确认，不作为核心结论 |
| 公开资料显示 | 可以使用，但不扩展未给出的结论 |
| 市场传闻 | 不作为核心事实，建议删除或提醒核查 |
| 有消息称 | 建议补充来源，或使用谨慎表达 |
| 业内认为 | 说明这是观点集合，不写成事实 |

### 8.3 特殊领域规则

| 领域 | 风险 | 处理规则 |
|---|---|---|
| 投融资 | 容易变成投资建议 | 可解读行业信号，不预测收益，不建议买卖 |
| 股价市值 | 数据时效性强 | 必须标注时间，不预测涨跌 |
| 政策监管 | 容易误读政策 | 建议引用原文，不替代法律解释 |
| 医疗健康 | 关系人身健康 | 不做诊疗、疗效、用药结论 |
| 法律争议 | 容易定性不当 | 使用“涉嫌”“争议”“有待确认”等表述 |
| 金融投资 | 合规风险高 | 不使用“稳赚”“必涨”“抄底”等表达 |

### 8.4 高风险表达替换规则

| 高风险表达 | 安全替换 |
|---|---|
| 一定会 | 可能会 |
| 必将 | 有望 / 可能 |
| 彻底改变 | 正在影响 |
| 证明了 | 释放出一个信号 |
| 全球第一 | 在相关榜单或公开信息中表现靠前 |
| 暴涨 | 明显上涨 / 阶段性上涨 |
| 颠覆行业 | 推动行业变化 |
| 最强 | 较具竞争力 |
| 唯一选择 | 一种可参考路径 |
| 保证收益 | 可能带来收益，但存在不确定性 |
| 无风险 | 风险相对较低 / 仍需结合实际情况判断 |
| 绝对领先 | 具有一定先发优势 |

### 8.5 必须提醒用户补充来源的情况

- 涉及金额、排名、市场份额、增长率，但没有来源。
- 涉及政策、监管、法律解释。
- 涉及医疗、投资、股价、收益预测。
- 原文只有观点，没有事实。
- 用户要求使用“第一、唯一、必然、稳赚”等强结论。
- 需要引用第三方报告但用户没有提供出处。
- 内容会影响企业声誉、公众判断或商业决策。

### 8.6 建议联网核查的情况

默认不联网，只基于用户提供资料写作。但出现以下情况时，应建议联网核查：

- 用户要求发布级内容，且涉及新闻事实、金额、政策、股价、融资、医疗、法律。
- 原文包含“据悉”“传闻”“知情人士”等不确定信息。
- 需要确认最新数据、最新政策、最新公告或最新市场变化。
- 文章中需要引用权威报告、榜单、财报、监管文件。

## 十一、用户交互设计

### 11.1 用户只给一段资讯时

AI 默认自动处理：

- 目标读者：行业从业者
- 文章用途：专业解读
- 写作风格：专业理性
- 字数范围：1000-1500 字，可按信息密度调整
- 输出内容：摘要、角度、标题、大纲、正文、风险提示等完整结果

### 11.2 用户给了资讯但没有说明风格时

默认使用“专业理性”。如果资讯明显偏企业发布，可自动偏“企业官方”；如果是热点，可自动偏“新闻解读 + 适度自媒体表达”；如果是投融资，可自动偏“财经科技分析”。

### 11.3 信息不足时的追问

最多追问 3 个问题，避免打断用户过多：

1. 这篇文章主要给谁看？
2. 你希望它更像“媒体解读”还是“企业官方发布”？
3. 是否有原始来源、公告链接或数据出处可补充？

### 11.4 用户说“不用问我，直接写”时

AI 使用默认配置，并在输出中注明：

```markdown
由于你要求直接生成，以下内容基于现有资料完成；涉及未提供来源的数据和判断，我会使用谨慎表达，并在风险提示中标出。
```

### 11.5 用户要求重写时

| 用户要求 | AI 操作 |
|---|---|
| 更爆款 | 加强标题钩子、开头冲突、节奏感，但不夸大事实 |
| 更专业 | 增加背景、数据、行业逻辑，降低情绪表达 |
| 更短 | 保留事实、观点、影响，删掉延展段 |
| 更像企业号 | 改为品牌官方语气，加入价值表达 |
| 更有观点 | 强化核心判断和论据，但保留边界 |
| 更适合朋友圈 | 缩短句子，增强共鸣和转发理由 |
| 更适合老板看 | 强化结论、机会、风险和行动建议 |

### 11.6 用户要求多个版本时

可按以下方式区分：

- 专业版：稳重、清晰、适合行业号。
- 传播版：更强开头、更强标题、更适合转发。
- 企业版：品牌立场明确，适合官方号。
- 短版：600-900 字，适合快讯。
- 深度版：1800-2500 字，适合趋势分析。

## 十二、质量评估标准

### 12.1 100 分评分表

| 项目 | 分值 | 评分标准 |
|---|---:|---|
| 信息准确性 | 15 | 事实无误，不编造数据，来源状态清楚 |
| 标题吸引力 | 10 | 有点击理由，但不夸大、不虚假承诺 |
| 结构完整度 | 10 | 开头、正文、结尾完整，逻辑顺畅 |
| 观点清晰度 | 10 | 核心判断明确，论据支撑充分 |
| 公众号阅读感 | 10 | 开头抓人，段落短，转场自然 |
| 传播性 | 10 | 有话题性、共鸣点、转发理由 |
| 专业度 | 10 | 背景、术语、行业逻辑准确 |
| 风险控制 | 10 | 区分事实和推测，规避敏感表达 |
| 语言流畅度 | 10 | 表达自然，无机械感、重复感 |
| 目标读者匹配度 | 5 | 内容深浅、语气和读者一致 |

### 12.2 分数解释

| 分数 | 解释 | 建议 |
|---|---|---|
| 90-100 | 可进入人工终审发布 | 检查标题和事实来源即可 |
| 80-89 | 质量较好 | 建议补充来源或微调标题 |
| 70-79 | 可用但需要润色 | 优化结构、标题和风险表达 |
| 60-69 | 风险或结构问题较多 | 建议重写核心段落 |
| 60 以下 | 不建议发布 | 重新拆解资讯和确认资料 |

## 十三、示例演示

### 13.1 原始资讯

```markdown
某 AI 办公软件公司宣布完成 5000 万美元 B 轮融资。本轮融资由某产业基金领投，公司称资金将用于产品研发、企业客户拓展和海外市场建设。公开资料显示，该公司主要提供 AI 文档生成、会议纪要和企业知识库产品。
```

### 13.2 AI 如何拆解资讯

```markdown
主体：某 AI 办公软件公司
事件：完成 5000 万美元 B 轮融资
资金用途：产品研发、企业客户拓展、海外市场建设
业务方向：AI 文档生成、会议纪要、企业知识库
事实信息：融资金额、轮次、资金用途、产品方向
需要核查：投资方名称、融资时间、公司真实业务数据
风险点：不能推断其一定会成为行业龙头，不能做投资建议
```

### 13.3 AI 如何判断文章类型

推荐类型：投融资解读型 + 行业分析型。  
推荐理由：资讯核心是融资事件，但更有公众号价值的角度是 AI 办公赛道的企业级应用趋势。

### 13.4 AI 如何选择文章角度

1. 从融资看 AI 办公软件进入企业级竞争阶段。
2. 为什么资本继续关注 AI 文档、会议纪要和知识库？
3. AI 办公产品下一阶段的关键，不只是生成内容，而是进入业务流程。

### 13.5 AI 如何生成标题

稳重专业型：

1. 某 AI 办公软件完成 B 轮融资，企业级应用竞争继续升温
2. 5000 万美元融资背后，AI 办公软件正在进入新阶段

财经科技分析型：

1. 从这轮融资看，AI 办公赛道的三个新变量
2. 资本继续下注 AI 办公，真正的竞争可能刚开始

趋势判断型：

1. AI 办公进入下半场：从效率工具走向企业场景
2. 文档、会议纪要、知识库之后，AI 办公还要解决什么？

### 13.6 AI 如何生成大纲

```markdown
导语：AI 办公赛道再次获得资本关注，但融资本身不是唯一重点。

一、发生了什么：公司完成 B 轮融资
- 交代融资金额、轮次、投资方和资金用途。

二、为什么是 AI 办公：企业效率需求仍在
- 解释 AI 文档、会议纪要、知识库的场景价值。

三、资金用途说明什么：研发、客户、海外三条线
- 分析企业级产品从工具走向系统的趋势。

四、行业竞争焦点：从单点工具转向企业知识系统
- 判断竞争重点可能从生成能力转向业务流程融合。

五、结尾：AI 办公的下一步，是能否真正进入业务流程
- 总结趋势，并引导读者讨论。
```

### 13.7 AI 如何说明风险点

- “5000 万美元”需确认来源。
- “领投方”需使用准确名称。
- 不写“必然上市”“将成为龙头”等确定性判断。
- 不写投资建议。
- 建议补充公司公告、投资方声明或权威媒体来源。
