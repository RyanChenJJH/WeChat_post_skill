# wechat-news-digest Skill v2.1 撰写阶段实施计划

> **给执行者：** 本计划把开发方案 v2.1 第十五章「实施清单」拆成可逐项执行的撰写任务。
> 详细内容规范以方案文档为准（同目录 `通过资讯生成公众号推文-Skill开发方案-v2.0.md`，当前内容为 v2.1），
> 本计划只负责"改哪个文件、写什么、按什么顺序、怎么验收"。步骤用 `- [ ]` 复选框跟踪。

**目标：** 把 `wechat-news-digest/` 这套 skill 实体文件从 v2.0 升级到 v2.1，落地 D8–D11 四项新决策
（选题评分 / 微信排版稿 / 分层节奏正文 / AI 内容标识）及配套机制。

**做法：** 三个新建 reference + 七个现有文件修改 + 一个不动；先建被依赖的底层文件，再改引用它们的总纲。
本阶段**只写 skill 文件，不跑 eval**（测试阶段另计）。

**技术栈：** 纯 Markdown skill 文件（官方三级渐进披露：SKILL.md + references/）。

## 全局约束（每个任务都隐含遵守）

- 术语统一：来源分级一律用 **S / A / B / C**（定义在 copyright-guardrails.md 9.2），其他文件只引用不另立。
- 评分口径：**废弃上游卡片 ⭐️ 评分**，选题一律走 topic-selection.md 的双门槛 + 三维模型。
- 三条铁律：版权 / 事实 / **标识与诚实**（新增）。AI 声明缺失在检查清单中为 ❌ 硬项。
- 排版稿：模型只产 markdown，HTML 由确定性转换产生——skill 文件里不要求模型手写 HTML 结构。
- 写作语言：简体中文；风格与现有 reference 一致（克制、可执行、解释"为什么"，少用全大写 MUST）。
