# wechat-news-digest · 微信公众号资讯速递 Skill

把公开资讯（可带原文的 md、新闻、公告、行业动态）整理成适合微信公众号发布的推文。
**版权安全是第一前提**：不洗稿、不照搬、不盗图、不冒充原创、永远注明来源，并按法规主动标识 AI 生成内容。

> 这是一个 [Claude Agent Skill](https://docs.claude.com/en/docs/agents-and-tools/agent-skills/overview)。
> 既可在 Claude / Codex 等支持 Skill 的客户端安装使用，也可"编译"成系统提示，在自建网页里调用第三方大模型 API。
> 本仓库内容由 AI 辅助开发，经人工把关。

---

## 它能做什么

- **选题**：多条资讯进来时，按"双门槛（版权/事实/定位一票否决）+ 三维读者价值评分"自动选题，**默认精选 2-3 条深写**（不沿用上游评分，要质不要量）。
- **撰写**：用**连贯叙事 + 通俗但克制**的口吻把每条讲透——把专业术语翻成大白话、补背景、打比方，让普通读者也能读懂、涨知识；每条 600–900 字。
- **版权风控**：从事实重写而非从表达重写；来源分级（S/A/B/C）、回溯一手源、**量化回查（与原文无连续 ≥13 字重合）**、独家内容处理、**英文原文不直译**。
- **事实安全**：区分事实/推测/观点，高风险领域（投融资/政策/医疗/法律/司法）措辞降级，不编造。
- **合规标识**：文末固定 AI 声明 + 提示勾选平台「AI 内容声明」，整理类内容不打"原创"标（依据《人工智能生成合成内容标识办法》）。
- **交付即用**：除纯文本正文外，另产**可直接粘贴进公众号编辑器的微信排版稿（内联样式 HTML）**。
- **发布前检查 + 变现建议**：逐项硬检查清单（任一 ❌ 不建议发布）+ 不污染可信度的低门槛变现建议。

## 设计取向（为什么这样做）

- **轻度整理，不做深度研报**：运营者非行业专家，让 AI 编"行业洞察"既无人背书、又最容易侵权出事。价值来自"把公开事实整理清楚 + 注明来源 + 标好风险"。
- **少而精**：宁可 2 条讲透，不要 5 条都浅。
- **丰富 ≠ 贴着原文展开**：写得通俗、能涨知识，但"丰富"必须来自 AI 自己的再解释，不是把原文展开复述——这样**更好读与更安全同时成立**。

## 安装与使用

### A. Claude / Codex 等支持 Skill 的客户端
- 方式一：安装打包文件 `wechat-news-digest.skill`。
- 方式二：把 `wechat-news-digest/` 目录放进客户端的 skills 目录。
- 然后把资讯（文本 / 链接 / 带原文的 md）发给它，说"整理成一期公众号推文 / 速递 / 轻解读"即可触发。

### B. 自建网页 + 第三方大模型 API
- 把 `SKILL.md` + 相关 `references/` 编译成系统提示，由应用按工作流分步编排调用；
- 选题、发布前检查作为网页 UI 卡点；
- 建议在应用层用代码实现两个确定性组件：**版权量化检查器**（13 字回查）与 **md→微信 HTML 转换器**。

### 输入格式（推荐：带原文 md）
每条资讯在摘要/背景/参考链接之外，附一个原文块即可让 skill 直接据原文撰写、无需联网：

```markdown
> [!quote]- 原文
>
> 来源：https://example.com/article
>
> （此处粘贴一手原文全文……）
```
